ABSTRACT
Retinal diseases like diabetic retinopathy (DR), age-related macular degeneration (AMD), and
glaucoma affect millions globally, often leading to preventable blindness if undetected early.
Traditional manual screening methods, while effective, face challenges such as subjectivity,
time-intensive workflows, and limited accessibility, particularly in underserved regions. To
address these gaps, this project explores the potential of advanced deep learning architectures—
Convolutional Neural Networks (CNNs), YOLO v8n, Vision Transformers (ViTs), and Swin
Transformers—to automate the detection and prediction of retinal pathologies through image
analysis. By leveraging standardized datasets (e.g., Messidor, Kaggle DR) and preprocessing
techniques like noise reduction and augmentation, the study systematically evaluates these
models across accuracy, precision, recall, and computational efficiency. CNNs serve as the
baseline for their proven success in hierarchical feature extraction, while YOLO v8n targets real-
time lesion localization. Vision Transformers and Swin Transformers, with their self-attention
mechanisms, capture global contextual patterns, offering state-of-the-art performance but
requiring greater computational resources. The societal urgency is underscored by rising diabetes
rates, with 629 million cases projected by 2045, nearly a third at risk for DR, necessitating
scalable solutions for early screening. Challenges include managing data heterogeneity (e.g.,
class imbalances, varying image quality) and balancing model accuracy with deployability in
low-resource settings. Innovations like hybrid CNN-Transformer ensembles and tools like Grad-
CAM for explainability aim to bridge clinical trust gaps. Future directions emphasize lightweight
models for mobile deployment, multimodal data integration (e.g., OCT + patient history), and
federated learning to enhance generalizability while preserving privacy. By comparing these
architectures, the project aims to establish a framework for equitable, AI-driven retinal
diagnostics, ultimately reducing preventable vision loss through accessible, cost-effective
screening tools..

2.1 WHAT IS CNN (Convolutional Neural Network)?
What is CNN?
Convolutional Neural Networks (CNNs) are a specialized type of deep learning algorithm
designed to process and analyse visual data, such as images and videos. They mimic the way the
human brain processes visual information, enabling machines to "see" and understand patterns in
visual inputs. CNNs have become the backbone of modern computer vision applications,
powering technologies like facial recognition, self-driving cars, and medical image analysis.
At its core, a CNN works by breaking down an image into smaller parts (pixels) and analysing
these parts to identify patterns. For example, a CNN can determine whether an image contains a
cat or a dog by learning the unique features of each, such as the shape of ears, fur texture, or eye
placement. This ability to automatically learn features from raw data makes CNNs incredibly
powerful and versatile.
How Does CNN Work?
CNNs process images step by step, extracting increasingly complex features at each stage.
Here’s how it works:
1. Image Input: The input to a CNN is usually an image represented as a grid of pixel
values (e.g., a 224x224 image with three colour channels for RGB).
2. Feature Extraction: The CNN uses filters (small matrices) to scan the image and detect
features like edges, corners, or textures. This process is called convolution.
3. Feature Mapping: The extracted features are stored in feature maps, which highlight
where specific patterns occur in the image.
4. Classification: After extracting features, the CNN uses fully connected layers to combine
them and classify the image into predefined categories (e.g., "cat" or "dog").
Key Components of CNN
1. Convolutional Layer: This is the heart of a CNN. It applies filters (kernels) to the input
image to detect specific features like edges or patterns. For example, one filter might
detect vertical edges, while another detects horizontal edges. Each filter produces an
output called a feature map.
2. Activation Function (ReLU): After applying filters, the activation function introduces
non-linearity into the model, allowing it to learn complex patterns.The most common
activation function is ReLU (Rectified Linear Unit), which replaces negative values with
zero.
3. Pooling Layer: Pooling reduces the size of feature maps while retaining important
information .For example, max pooling takes the maximum value in a small region (e.g.,
2x2), preserving the most prominent features.
5

=== PAGE 14 ===
4. Fully Connected Layer: This layer combines all extracted features and makes
predictions based on them. It connects every neuron in one layer to every neuron in the
next layer.
5. Dropout Layer:Dropout randomly disables some neurons during training to prevent
overfitting and improve generalization.
Why Are CNNs Important?
CNNs are revolutionary because they automate feature extraction, eliminating the need for
manual engineering of features like edges or textures. They can learn directly from raw data
without requiring domain-specific knowledge. This makes them highly adaptable for various
tasks across industries.
For example:
 In facial recognition systems, CNNs can learn to identify unique facial features like eyes,
nose shape, or jawline.
 In medical imaging, they can detect abnormalities such as tumours or lesions without
requiring explicit instructions.
Applications of CNNs
1. Image Classification: Assigning labels to images (e.g., identifying whether an image
contains a cat or dog). Example: Google Photos uses CNNs to automatically organize
pictures based on their content.
2. Object Detection: Identifying objects within an image and drawing bounding boxes
around them. Example: Self-driving cars use CNNs to detect pedestrians, vehicles, and
traffic signs.
3. Medical Imaging: Analysing X-rays, MRIs, or retinal images to detect diseases like
cancer or diabetic retinopathy. Example: AI-powered systems can assist radiologists by
highlighting suspicious areas in medical scans.
4. Natural Language Processing (NLP): While primarily used for images, CNNs can also
process text data for tasks like sentiment analysis or document classification.
How Do CNNs Learn?
CNNs learn through a process called supervised learning, The model is trained on labelled data
(e.g., images with labels like "cat" or "dog"). During training, it adjusts its internal parameters
(weights) based on how well it predicts the correct label. The goal is to minimize the error
between predicted labels and actual labels using optimization techniques like gradient descent.
Over time, the model improves its ability to recognize patterns and generalize to new data it
hasn’t seen before.
Hierarchical Learning in CNNs
CNNs learn features hierarchically:
 Early Layers: Detect simple patterns like edges or corners.
 Middle Layers: Combine these simple patterns into more complex structures like shapes
or textures.
 Final Layers: Recognize high-level objects like faces or animals.
6

=== PAGE 15 ===
This hierarchical approach mirrors how humans perceive visuals—starting with basic details
before identifying the complete object.
Advantages of CNNs
1. Automatic Feature Extraction: Unlike traditional methods that require manual feature
engineering, CNNs learn directly from raw data.
2. Translation Invariance:They can recognize objects regardless of their position in an
image.
3. High Accuracy:CNNs often outperform traditional methods in tasks like object
recognition and medical diagnosis.
Challenges of CNNs
1. Computational Costs: Training large models requires powerful GPUs and significant
memory resources.
2. Black Box Nature: It’s difficult to understand exactly how a CNN makes decisions.
3. Overfitting:If not trained properly, CNNs may memorize training data instead of
generalizing well to new data.
Popular Architectures in CNNs
1. LeNet-5 (1998): One of the earliest CNN models used for digit recognition (e.g., reading
handwritten numbers).
2. AlexNet (2012): Revolutionized computer vision by achieving breakthrough accuracy
levels on large-scale datasets.
3. ResNet (2015):Introduced residual connections that allow training very deep networks
without performance degradation.
Future Directions for CNNs
1. Lightweight Models for Edge Devices: Models like MobileNet are optimized for
smartphones and IoT devices.
2. Explainable AI (XAI): Tools like Grad-CAM help visualize which parts of an image
influenced a CNN’s decision.
3. Integration with Transformers:Combining CNNs with transformers opens new
possibilities for video analysis and multimodal learning.
Convolutional Neural Networks (CNNs) have transformed how machines perceive visual data by
automating feature extraction and enabling hierarchical learning. They have found applications
across diverse fields such as healthcare, autonomous vehicles, and entertainment while
continuing to evolve with advancements in lightweight architectures and explainability tools.
Despite challenges like computational costs and interpretability concerns, their impact on
artificial intelligence remains unparalleled paving the way for smarter systems capable of
understanding and interacting with the world visually.

Flowchart:
Retinal image preprocessing standardizes input data for convolutional neural networks (CNNs)
through resizing and normalization. Resizing to 224×224 pixels ensures compatibility with
common architectures like VGG16 and ResNet while balancing computational efficiency and
feature retention. For datasets such as RFMiD, which include high-resolution images (e.g.,

and minimizing redundant computation. This pipeline optimizes memory usage (critical for
systems with 16GB RAM) and ensures consistent batch processing. The combined steps mitigate
gradient instability, reduce training time, and preserve diagnostically relevant features for
accurate classification.

4. CNN Architecture
Model Type: Vanilla CNN (Basic Sequential Model), Layers:
model = Sequential([
Conv2D(32, (3,3), activation='relu', input_shape=(224,224,3)), # Feature extraction
MaxPooling2D(2,2), # Downsampling
Conv2D(64, (3,3), activation='relu'),
MaxPooling2D(2,2),

The Vanilla CNN architecture employed for our project is a basic sequential model designed for
binary classification tasks. It is constructed using the Keras Sequential API and consists of
several key layers. The model begins with three Conv2D layers, each followed by
a MaxPooling2D layer, to progressively extract hierarchical features from the input images and
reduce spatial dimensions. Each Conv2D layer uses a 3x3 kernel and ReLU activation to capture
complex patterns like edges, textures, and lesions. The MaxPooling2D layers downsample the
feature maps while retaining critical information.

3.1 WHAT IS YOLO V8n?
Imagine you’re tasked with scanning a crowded room to instantly identify and locate specific
individuals, such as someone wearing a red shirt or carrying a bag. You need to do this quickly
and efficiently without missing anyone. This is exactly how YOLOv8n operates in the realm of
computer vision—it’s a lightweight, fast, and efficient object detection model designed for real-
time tasks. YOLO stands for You Only Look Once, and the "v8n" version is the nano variant of
YOLOv8, optimized for devices with limited computational power, such as smartphones or edge
devices.
YOLOv8n processes an image in one pass, analysing it holistically rather than scanning it
multiple times like older methods. It divides the image into a grid and checks each cell for
objects. For example, in medical imaging, it might scan retinal images to detect abnormalities
like microaneurysms or haemorrhages
Look Once, Detect Everything: Unlike older systems that scan images multiple times, YOLOv8n
analyses the entire image in one glance (hence "You Only Look Once"). It divides the image
into a grid and checks each cell for objects, like how you’d scan a room section by section.
Three Key Parts:
Backbone (The Observer): Uses a lightweight neural network called CSPDarknet to quickly
identify basic patterns (edges, shapes). Think of it as your peripheral vision catching movement.
Neck (The Analyst): Combines details from different scales using PANet, like zooming in/out
to see both small lesions and large structures in a retinal scan.
Head (The Decider): Predicts what the object is (e.g., "hemorrhage") and draws a box around it.
No pre-defined boxes (anchor-free)—it guesses shapes freely, like sketching on the fly.

=== PAGE 23 ===
Training Tricks: Mix & Match Images: During training, it practices collage-like images
(mosaic augmentation) to handle cluttered scenes, similar to studying in a noisy café to improve
focus. Learn from Mistakes: Uses focal loss to prioritize hard-to-spot objects (e.g., tiny
lesions), just like a teacher spending extra time on tricky topics.
Why does YOLOv8n Stand Out?
Speed Demon: Processes ~250 images/second on a laptop—faster than you blink! Ideal for real-
time tasks like monitoring video feeds.
Lightweight: At just 3 million parameters (vs. 86 million in larger models), it’s like a
smartphone app, not a supercomputer.
Accuracy on a Diet: Achieves ~90% precision in detecting objects despite its small size,
rivaling bulkier models.

4.1 WHAT IS ViT (VISION TRANSFORMERS)?
What is Vision Transformer (ViT)?
Imagine you’re solving a jigsaw puzzle. Instead of focusing on each piece in isolation, you look
at how pieces fit together—how the colors, edges, and patterns connect to form the bigger
picture. This is how Vision Transformers (ViTs) approach image analysis: they don’t just focus
on individual parts of an image but also understand how those parts relate to one another across
the entire image. ViTs are a revolutionary type of deep learning model that bring the power of
transformers (originally designed for natural language processing) into the world of computer
vision.
How Does ViT Work?
1. Breaking Down the Image into Patches
Instead of processing an image pixel by pixel, ViT divides it into smaller patches, like cutting a
photo into equal squares. For example, a 224x224-pixel image might be split into 16x16 patches,
resulting in 196 patches in total. Each patch is treated as an individual "word" in the language of
images.
This approach allows ViT to treat images like sentences in a language model, where each patch
represents a distinct piece of information.
2. Converting Patches into Embeddings
Once the image is divided into patches, each patch is flattened (converted into a 1D array) and
passed through an embedding layer. This embedding layer transforms each patch into a
numerical representation that captures its visual features, such as colors, textures, and edges.
Think of this step as translating each puzzle piece into a unique code that describes its shape and
color.
25

=== PAGE 34 ===
3. Adding Positional Information
To ensure that the model knows where each patch belongs in the original image, positional
embeddings are added to the patch embeddings. This is crucial because without positional
information, the model wouldn’t know if a patch belongs to the top-left corner or the center of
the image.
In medical imaging, such as retinal disease detection, knowing where a lesion or abnormality is
located (e.g., near the optic disc or macula) is critical for diagnosis.
4. Self-Attention Mechanism
The heart of ViT lies in its self-attention mechanism, which allows it to analyze relationships
between all patches in an image. For example: A hemorrhage patch might "pay attention" to
nearby exudates. The optic disc region might influence how the model interprets abnormalities in
surrounding areas.
How Self-Attention Works:Each patch compares itself to every other patch in the image and
assigns importance scores (attention weights). This helps the model understand how different
parts of the image interact with one another.
Analogy:
It’s like solving a mystery by connecting clues scattered across different parts of a crime scene.
5. Classification Head
After processing all patches through multiple layers of self-attention and feed-forward networks,
ViT aggregates all this information into a single output. For retinal disease detection, this output
could be: A binary classification: "Disease" or "Non-disease." A multi-class classification:
Specific conditions like diabetic retinopathy or glaucoma.
Why Vision Transformers Are Revolutionary
ViTs have introduced several advantages over traditional Convolutional Neural Networks
(CNNs):
1. Global Context Awareness: Unlike CNNs, which focus on local features (e.g., edges
or textures), ViTs analyse relationships across the entire image. This makes them

specific tasks with relatively small datasets, making them versatile for applications like

5.1 WHAT IS SWIN?
Imagine you’re looking at a city from above. First, you focus on neighborhood streets, houses,
and parks. Then, you zoom out to see how this neighborhood connects to the larger city,
observing highways, bridges, and rivers. Finally, you zoom out even further to understand how

structures is exactly how the Swin Transformer processes images. It’s a groundbreaking deep
learning model that combines the strengths of traditional Convolutional Neural Networks
(CNNs) and Vision Transformers (ViTs), enabling machines to analyze images with both
precision and context.
How Swin Transformer Works: The Basics
1. Breaking Images into Windows
The Swin Transformer starts by dividing an image into smaller "windows," much like splitting a
photo into tiles. Each window contains a fixed number of pixels (e.g., 4x4 or 7x7), and the model
processes these windows individually. This approach allows Swin to focus on local details, such
as edges or textures within each window.
Why Windows?
Windows make it easier for the model to focus on small regions of an image while reducing
computational complexity compared to analyzing the entire image at once.
2. Hierarchical Structure
After processing individual windows, Swin Transformer organizes them hierarchically. It starts
with small windows to capture fine details (like blood vessels in a retinal image) and gradually
merges them into larger windows to understand broader patterns (like macular edema or optic
32

=== PAGE 41 ===
disc abnormalities). Think of this as assembling a puzzle—starting with individual pieces and
gradually connecting them to form the complete picture.
3. Shifted Windows
One of Swin’s most innovative features is its shifted window mechanism. Instead of analyzing
the same windows repeatedly, it shifts them slightly between layers. This ensures that
overlapping areas between windows are also considered, allowing the model to capture
relationships between neighboring regions. In medical imaging, such as retinal scans,
abnormalities often span multiple regions (e.g., hemorrhages near blood vessels). Shifted
windows help Swin Transformer connect these regions seamlessly.
4. Self-Attention Within Windows
Within each window, Swin applies self-attention, a mechanism that assigns importance scores to
different pixels based on their relationships. For example: In a retinal image, self-attention might
prioritize pixels representing lesions while ignoring background noise. In an aerial photo, it
might focus on roads and buildings rather than empty fields.
Self-attention allows Swin Transformer to understand which parts of an image are most relevant
for making predictions.
5. Patch Merging
As Swin processes deeper layers, it merges patches from neighboring windows into larger ones.
This hierarchical merging enables the model to transition from local analysis (small details) to
global understanding (overall patterns).
Example:
In retinal disease detection, Early layers might identify microaneurysms or exudates.Later layers
might analyze how these features interact across the retina to predict conditions like diabetic
retinopathy or glaucoma.

7.3 CHALLENGES
1. Data Leakage Issue: Data leakage occurs when information from the test set
unintentionally influences the training process, leading to overly optimistic performance
metrics. This can result in the model performing well during training but failing to
generalize to unseen data. Proper data splitting (train/validation/test) and ensuring no
overlap between datasets are critical to mitigate this issue.
2. Overfitting Due to Variable Data: The large number of variables and features in retinal
images can cause the model to memorize the training data instead of learning
generalizable patterns. Overfitting leads to high accuracy on the training set but poor
performance on new data. Techniques like data augmentation (e.g., flipping, rotation),
dropout layers, and weight decay are used to counteract this problem.
71

3. Model Memorization: Sometimes, the model achieves unnaturally high accuracy, such as
100%, by memorizing the training data rather than learning meaningful features. This is a
sign of overfitting and indicates that the model is not learning in a robust manner.
Regularization techniques, cross-validation, and early stopping are essential to ensure
genuine learning.
4. Unavailability of GPU for Training Large Datasets: Training large datasets like RFMiD
on a CPU significantly increases computation time, making it impractical for deep
learning models such as Swin Transformers. GPUs or TPUs are necessary for efficient
training, but their unavailability can hinder progress. Cloud-based platforms like Google
Colab or AWS can provide temporary GPU access as a workaround.
5. Long Training Times Due to Large Epochs: Training models over many epochs is often
required for convergence, but it drastically increases computational time, especially for
complex architectures like Swin Transformers. Learning rate schedulers and batch size
optimization can help reduce training time without compromising performance.
6. Missing Files in Image Folders: Missing or corrupt image files in the dataset can cause
crashes during training or evaluation. Implementing error-handling mechanisms in the
data loader ensures that such files are skipped without interrupting the process.
7. Regularization Requirements: Regularization is essential to prevent overfitting and
ensure that models generalize well to unseen data. Techniques like dropout, weight
decay, and batch normalization are frequently employed during training to maintain
stability and improve robustness.
Addressing these challenges requires careful dataset preparation, robust model design, and
efficient resource management. By implementing strategies like regularization, error handling,
and leveraging cloud-based GPUs, these issues can be mitigated effectively, ensuring reliable
and accurate retinal disease detection models.

7.6 GLOSSARY
Vision Transformer (ViT): A deep learning model that processes images by dividing them into
patches and analysing their relationships using self-attention mechanisms. ViT excels in
capturing global context, making it ideal for complex tasks like retinal disease detection.
Swin Transformer: A hierarchical model that processes images using shifted windows to
capture both local features and global patterns efficiently. It is particularly effective for high-
resolution images and multiclass disease classification tasks.
YOLOv8n:A lightweight object detection model designed for real-time applications. YOLOv8n
detects and localizes lesions in retinal images by providing bounding boxes with confidence
scores, making it suitable for lesion localization alongside classification.
Convolutional Neural Network (CNN): A traditional deep learning model that extracts
localized features through convolutional layers and performs binary classification (Disease/Non-
disease). While efficient, CNNs struggle with capturing long-range dependencies compared to
transformers.
Data Augmentation: Techniques like resizing, flipping, rotation, and color jittering applied to
training data to improve model robustness and prevent overfitting.
Regularization: Methods such as dropout, weight decay, and batch normalization used to
stabilize training and reduce overfitting.
Shifted Window Mechanism: A unique feature of Swin Transformers that shifts window
boundaries between layers to ensure overlapping regions are analyzed, improving context
understanding across image patches.
Confusion Matrix: A tool used to evaluate model performance by visualizing true positives,
false positives, true negatives, and false negatives across classes.
Evaluation Metrics:
Accuracy: Percentage of correct predictions out of total predictions.

=== PAGE 83 ===
Precision: Proportion of true positive predictions among all positive predictions.
Recall: Proportion of true positive predictions among all actual positives.
F1-Score: Harmonic mean of precision and recall.
Mixed Precision Training (fp16): A training technique that uses half-precision floating points
to reduce memory usage while maintaining performance, particularly useful for ViTs and Swin
Transformers.
GPU Constraints: The need for high-performance GPUs to train large models like ViTs and
Swin Transformers efficiently due to their computational demands.
